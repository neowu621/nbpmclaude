Agentic 工作流完整指南

當 AI 從「回答問題」進化到「完成任務」，PM 的角色也從「提問者」變成「委託人」。 這一章教你像管理一個外包團隊一樣管理 AI 代理。

什麼是 Agent？用 PM 的語言說

傳統的 AI 對話：你問一句，它答一句。
Agentic 的 AI：你給一個目標，它自己拆解步驟、使用工具、檢查結果、修正錯誤，直到完成。

組成一個 Agent 的三個零件：

  • 模型（大腦）：負責理解目標、規劃步驟、判斷下一步。
  • 工具（手腳）：讀寫檔案、搜尋網頁、執行程式、透過 MCP（Model Context Protocol，一種讓 AI 連接外部系統的開放標準）存取 Jira、Notion、Slack、資料庫等。
  • 迴圈（工作方法）：探索 → 規劃 → 執行 → 驗證，不斷循環直到達成目標或需要人類介入。

代理循環：Explore → Plan → Execute → Verify

以「整理本季所有客訴並產出洞察報告」為例，Agent 的實際運作：

  1. Explore 探索：先讀取資料夾，發現有 8 個 CSV 檔、格式不一致、其中兩個檔案有重複資料。
  2. Plan 規劃：提出計畫——統一格式 → 去重 → 依問題類型分類 → 統計 → 產出報告。（好的實踐：在這裡停下來讓你確認）
  3. Execute 執行：逐步執行，過程中發現新問題（如某欄位缺值）會自行調整或回報。
  4. Verify 驗證：檢查總筆數是否對得上、分類是否有遺漏，產出處理摘要。

PM 的治理三件事

1. 權限設計：最小必要原則

權限層級範例適用
唯讀讀取數據、搜尋文件分析與研究類任務，預設起點
可寫（沙盒）在指定資料夾產出報告產出類任務，影響範圍受控
可寫（正式）更新 Jira 工單、寄送郵件必須搭配人工檢核點

2. 檢核點：不可逆的動作之前必須有人

  • 寄出任何對外訊息之前
  • 修改正式文件、工單狀態之前
  • 刪除或覆蓋資料之前

口訣：「可逆的放行，不可逆的把關。」

3. 驗收定義：像寫 acceptance criteria 一樣寫任務

任務：整理 Q2 客訴資料並產出洞察報告。
完成的定義：
1. 所有 CSV 已合併去重，處理筆數與原始筆數的差異有說明
2. 分類使用我們的既有分類法（附件），不要自創類別
3. 報告含：各類別佔比、環比變化、Top 3 惡化項目與證據
4. 過程中遇到無法分類的資料，列出來問我，不要猜

常見問題

Q：PM 不會寫程式，能用 Agentic 工具嗎？

能。Claude Code 這類工具用自然語言下指令即可；你的價值在於「規格清楚、驗收嚴謹」——這比會寫程式更關鍵。

Q：Agent 會不會做出我不想要的事？

權限與檢核點就是為此存在。從唯讀任務開始建立信任，逐步放寬——跟你帶新人的方式一樣。

Q：什麼任務「不適合」交給 Agent？

目標模糊、判斷密集、政治敏感的任務。「幫我決定要不要跟這個客戶簽約」不是 Agent 任務；「把這個客戶過去一年的往來紀錄整理成時間軸」才是。
