情境：你委託 Claude Code「分析本季客訴資料並產出洞察報告」。 按「下一步」，看 Agent 在每個階段實際做什麼、以及 PM 在哪裡介入。

按下「開始模擬」，觀察 Agent 的工作循環。

看完模擬，記住三件事

  • Plan 階段是 PM 最重要的介入點——在 Agent 動手前確認方向，成本最低。
  • Execute 過程中 Agent 會遇到意外（缺值、格式不一致），好的任務描述會告訴它「遇到 X 就問我，不要猜」。
  • Verify 不能省——要求處理摘要（處理幾筆、跳過幾筆、為什麼），這是你驗收的依據。
