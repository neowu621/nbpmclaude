模式診斷 🧭

想著你手上的一個真實任務，回答三個問題，我們幫你判斷該用哪種模式。

Q1. 這個任務需要多少「公司或產品的內部脈絡」？

Q2. 這是一次性的，還是會反覆做？

Q3. 產出的形式是什麼？

診斷邏輯說明

  • 需要大量資料處理或檔案產出 → 優先考慮 Code
  • 需要內部脈絡或反覆進行 → 建立 Project，用 Cowork
  • 通用知識、一次性、純文字 → Chat 最快

完整說明見 三種工作模式指南。
