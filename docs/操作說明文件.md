# Claude PM 入門手冊 — 操作說明文件

> 網站：`https://nbpmclaude.zeabur.app`
> 版本：v1.0（2026-07-02）
> 技術：純靜態網站（HTML / CSS / Vanilla JS），無後端、無資料庫、無追蹤

---

## 一、網站定位

本網站是一本為**產品經理（PM）**設計的 Claude 入門互動手冊。
架構參考教育領域 Claude 教學手冊的學習方法論（概念 → 引導 → 實作 → 資源），
內容全部以 PM 的工作情境重新編寫：需求規格、用戶研究、數據指標、
利害關係人溝通、市場競品、規劃執行。

**設計原則：**
1. **AI 產出，PM 判斷** —— 全站任務都強調人類保留最終判斷權
2. **心智模型優先** —— 概念比功能截圖耐用，功能細節導向官方文件
3. **即學即用** —— 所有 Prompt 一鍵複製，任務標注時間與難度

## 二、訪客使用指南

### 2.1 建議學習路徑

| 對象 | 路徑 | 預估時間 |
|---|---|---|
| 完全新手 | 首頁 → 概念學習 1–4 章 → 模式診斷 → 任務 T10 → 概念 5–8 章 | 2 小時 |
| 已有 AI 經驗 | 4D 自評 → 補最弱維度的章節 → 2 個進階任務 | 1.5 小時 |
| 帶團隊導入的主管 | 概念全章 → 案例 5 → 團隊協作工具包 | 2 小時 |
| 做 AI 功能的 PM | 詞彙表 → 第 5 章 Agentic → 官方開發者文件 | 1 小時 |

### 2.2 互動功能操作

| 功能 | 位置 | 操作方式 |
|---|---|---|
| 複製 Prompt | 全站深色程式碼區塊 | 點右上角「複製」→ 貼到 Claude → 替換【】占位字 |
| 任務篩選 | 任務實作頁 | 點「工作領域／難度／模式」按鈕即時過濾；點任務標題展開內容 |
| 模式診斷 | 引導式學習 → 模式診斷 | 回答 3 題 → 按「診斷結果」 |
| Prompt 組裝器 | 引導式學習 → Prompt 組裝器 | 填 5 個欄位 → 「組合 Prompt」→ 一鍵複製 |
| 4D 自評 | 引導式學習 → 4D 自評 | 拖 4 個滑桿 → 「看評估結果」→ 取得最弱維度建議 |
| Agentic 模擬器 | 引導式學習 → 模擬器 | 連按「下一步」走完 4 階段，注意 2 個 PM 介入點 |

### 2.3 注意事項

- 本站**不儲存任何使用者資料**，互動工具的結果重新整理即消失
- 建立 Claude 個人帳號需**年滿 18 歲**
- 把公司資料貼入 AI 前，請先讀「概念學習・第 8 章」與團隊資料分級規範

### 3.2 本機預覽

不需要任何建置工具，兩種方式擇一：

```bash
# 方式一：直接用瀏覽器開 index.html（雙擊即可）

# 方式二：起一個本機伺服器（建議，路徑行為與正式站一致）
cd nbpmclaude
python -m http.server 8080
# 或
npx serve .
```

### 3.3 修改內容的常見場景

| 想改什麼 | 改哪裡 |
|---|---|
| 全站配色／字體 | `assets/style.css` 開頭的 `:root` CSS 變數 |
| 新增一個任務 | 複製 `tasks/index.html` 中任一 `<details class="task">` 區塊，改 `data-role / data-level / data-mode` 屬性即可被篩選器識別 |
| 新增一條 Prompt | `resources/prompts.html` 複製一組 `<h4>` + `.prompt-box` |
| 新增詞彙 | `resources/glossary.html` 複製一個 `.glossary-item` |
| 改導覽列 | 每頁的 `<nav>` 都是獨立的（純靜態），需逐頁修改；子目錄頁面注意相對路徑 `../` |

### 3.4 部署流程（GitHub → Zeabur）

網站為純靜態站，Zeabur 綁定 GitHub repo 後自動部署：

1. **推送更新**：
   ```bash
   git add -A
   git commit -m "更新內容"
   git push origin main
   ```
2. **自動部署**：Zeabur 偵測到 main 分支更新後自動重新部署（約 1 分鐘）
3. **網域**：服務網域設定為 `nbpmclaude.zeabur.app`（Zeabur 控制台 → 服務 → Domains）

首次部署設定（僅需一次）：
1. 登入 [Zeabur](https://zeabur.com) → Create Project
2. Add Service → GitHub → 選擇本 repo
3. Zeabur 自動辨識為靜態網站（Static）並完成部署
4. 在服務的 **Networking / Domains** 加入子網域 `nbpmclaude` → 產生 `nbpmclaude.zeabur.app`
